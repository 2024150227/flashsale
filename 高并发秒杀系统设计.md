## 🧩 系统核心功能

| 功能         | 描述                                          |
| ------------ | --------------------------------------------- |
| **商品展示** | 查看秒杀商品列表及详情（含剩余库存）          |
| **秒杀下单** | 用户点击“立即秒杀”，尝试创建订单              |
| **异步处理** | 下单请求不直接写 DB，而是发到 Kafka           |
| **订单消费** | 后台消费者从 Kafka 取消息，真正扣库存、建订单 |
| **限流保护** | 对秒杀接口做 QPS 限制（如 1000 req/s）        |
| **防超卖**   | 通过 Redis Lua 脚本保证库存原子性             |

## 🔑 关键技术实现细节（你必须掌握的部分）

### 1. **Redis 缓存商品 & 库存预热**
- 启动时将商品信息（ID、名称、库存）加载到 Redis Hash。
- 商品详情接口优先读 Redis，缓存失效再查 DB 并回填。
- **库存字段单独存在 Redis String 中**（如 `stock:product:1001`），便于 Lua 操作。

```python
# utils/redis_client.py
import redis
redis_client = redis.StrictRedis(host='redis', port=6379, decode_responses=True)

# 预热库存
redis_client.set(f"stock:product:{product_id}", initial_stock)
```

---

### 2. **Redis Lua 脚本 —— 防超卖核心！**
这是面试官必问点！必须用 Lua 保证“读库存 → 判断 >0 → 减1”是原子操作。

```lua
-- lua/decrease_stock.lua
local stock_key = KEYS[1]
local current = tonumber(redis.call('GET', stock_key))
if current and current > 0 then
    redis.call('DECR', stock_key)
    return 1  -- 成功
else
    return 0  -- 失败（库存不足）
end
```

在 Python 中调用：

```python
# services/order_service.py
lua_script = """
... 上面的 Lua 代码 ...
"""
decrease_stock = redis_client.register_script(lua_script)

# 执行
result = decrease_stock(keys=[f"stock:product:{product_id}"])
if result == 1:
    # 库存扣减成功，可以发 Kafka
else:
    raise Exception("库存不足")
```

---

### 3. **Kafka 异步下单**
- 用户点击秒杀 → FastAPI 接口只做：**验证 + 限流 + 发 Kafka 消息** → 立即返回 `{ "status": "queued" }`。
- 消息格式示例：
  ```json
  { "user_id": 123, "product_id": 1001, "timestamp": "2026-04-26T23:30:00Z" }
  ```

```python
# utils/kafka_producer.py
from confluent_kafka import Producer

producer = Producer({'bootstrap.servers': 'kafka:9092'})

def send_order_message(message: dict):
    producer.produce('flashsale_orders', value=json.dumps(message))
    producer.flush()
```

---

### 4. **独立消费者服务（后台进程）**
- 单独写一个 `consumer.py`，用多线程或 asyncio 消费 Kafka。
- 消费逻辑：
  1. 收到消息
  2. 执行 Redis Lua 扣库存
  3. 如果成功 → 写 MySQL 订单表
  4. 如果失败 → 记录日志（用户秒杀失败）

> ✅ **幂等性保障**：即使 Kafka 重复投递，因为 Redis Lua 是幂等的（第二次扣会失败），所以不会重复下单。

### 5. **限流（Rate Limiting）**
用 `slowapi` 库（基于令牌桶）对 `/orders/` 接口限流。

```python
# main.py
from slowapi import Limiter, _rate_limit_exceeded_handler
from slowapi.util import get_remote_address

limiter = Limiter(key_func=get_remote_address)
app.state.limiter = limiter
app.add_exception_handler(RateLimitExceeded, _rate_limit_exceeded_handler)

@app.post("/api/v1/orders/")
@limiter.limit("1000/minute")  # 每分钟最多1000次
async def create_order(...):
    ...
```

### 面试可能被问的问题：
- **Q：为什么用 Kafka 不直接写 DB？**  
  A：瞬时高并发下直接写 DB 会导致连接池耗尽、死锁。Kafka 作为缓冲，让消费者以可控速度处理，保证系统稳定性。

- **Q：Redis 挂了怎么办？**  
  A：有降级策略——如果 Redis 不可用，直接走 DB（加分布式锁），但性能会下降。同时监控告警，快速恢复。

- **Q：Lua 脚本怎么防止超卖？**  
  A：因为 Redis 是单线程执行 Lua 脚本，整个“判断+扣减”过程不可分割，避免了并发竞争。
